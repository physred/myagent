# 知识问答报告生成

## 使用场景
当用户要求生成报告、总结问答内容、整理问答结果，或者要求把知识问答过程输出为文档时，按照本技能执行。

## 输出目标
在 workspace 下生成一个 Markdown 报告文件，文件名建议为 `qa_report_YYYYMMDD_HHMM.md`。

## 生成流程
1. 先确认报告主题、问答范围、时间范围或会话范围。
2. 汇总问答中的核心问题、结论、引用来源、待补充项。
3. 将报告写入 workspace 下的 `.md` 文件。
4. 返回生成的文件路径给用户。

## 报告格式
报告必须包含以下章节，顺序固定：

### 1. 标题
- 使用 `#` 作为一级标题。
- 标题格式建议为：`知识问答报告 - {主题}`。

### 2. 基本信息
- 生成时间
- 主题
- 来源会话或数据范围
- 生成者

### 3. 问题摘要
- 用列表列出本次报告覆盖的主要问题。

### 4. 问答正文
- 每个问题单独成节，格式如下：
  - `## 问题 {序号}`
  - 问题内容
  - 回答内容
  - 如有引用，列出引用来源

### 5. 结论与建议
- 汇总整体结论。
- 给出后续建议或待处理事项。

### 6. 附录
- 如有需要，放置额外说明、原始片段或补充材料。

## 写入要求
- 使用 Markdown 格式。
- 内容尽量简洁、结构清晰。
- 如果信息不足，明确标注“待补充”，不要编造。
- 报告必须保存到 workspace 目录下，不能写到 workspace 外。
